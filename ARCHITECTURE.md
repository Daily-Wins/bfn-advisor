# Architecture

## Overview
K2K3.ai is a SvelteKit RAG app that answers questions about Swedish accounting regulations (BFN/K2/K3/BRF/Årsbokslut/etc.) with exact paragraph citations. Source regulations are extracted to markdown chunks under `app/knowledge/`, retrieved per request via a hybrid keyword + semantic router, and passed as grounded context to an OpenRouter LLM.

## Tech stack
- Framework: SvelteKit 2 (Svelte 5 runes)
- Hosting: Vercel (`@sveltejs/adapter-vercel`)
- DB: Turso (libSQL) via `@libsql/client`
- Auth: Auth0 via `@auth/sveltekit`
- LLM: OpenRouter — `anthropic/claude-sonnet-4.6` for answers, `baai/bge-m3` for embeddings
- Styling: Tailwind CSS 4

## Request flow (happy path: POST /api/chat)
1. `src/hooks.server.ts` runs the Auth0 handler, then sets `locals.user` (registered) or assigns an `anon_id` cookie and loads the anonymous session question count.
2. `src/routes/api/chat/+server.ts` enforces the freemium gate (anonymous users are blocked after 5 questions with `limit_reached`).
3. Builds a routing query from the last 2 prior user messages plus the current message.
4. `resolveScope(regulation)` classifies the UI-supplied regulation filter as `single`, `comparison` (`K2K3`), or `auto`.
5. `routeQuestionSemantic(query)` in `src/lib/server/semantic-router.ts`:
   - Layer 1: `matchPunktDirect` — regex for `punkt X.Y` → direct chapter hit (K2 only).
   - Layer 2 (parallel): `getSemanticResults` embeds the query via OpenRouter bge-m3 and ranks sections by cosine similarity (with regulation-boost ×1.3 / ×0.7); `routeQuestion` from `router.ts` does keyword scoring with regulation boost.
   - Both lists fused via Reciprocal Rank Fusion (k=60), deduped by `dir/file`. Falls back to keyword-only on embedding errors. Returns top 4.
6. `filterMatchesByScope(matches, scope)` restricts to the selected regulation(s); falls through to unfiltered results if no match survives.
7. `loadChapters(matches)` reads markdown from `app/knowledge/<dir>/<file>`, capped at 80 000 chars total (`MAX_CHARS`).
8. `formatContext` prepends verified `belopp` facts and changed-rule notes from `facts.json`, then concatenates chapter bodies with `--- START / SLUT ---` delimiters.
9. `streamCompletion(context, messages, regulation)` in `ai.ts` builds a regulation-specific system prompt (`buildSystemPrompt`) and streams Claude's response via OpenRouter (`stream: true`, `temperature: 0`, `max_tokens: 2048`).
10. Usage recorded: anonymous → `incrementAnonymousCount`; registered → `recordUserQuestion`.
11. Upstream SSE is re-parsed and re-emitted to the client as `data: {content}` events, with a final `{done, sources}` frame.

## Data
- `app/knowledge/<regulation>/*.md` — authoritative chapter markdown loaded at runtime. Regulations present: `arsbokslut`, `bokforing`, `brf-upplysningar`, `fusioner`, `gransvarden`, `k1-enskilda`, `k1-ideella`, `k2-arsredovisning`, `k3-arsredovisning`.
- `app/src/lib/server/facts.json` — structured verified amounts and changed-rule notes per chapter (generated by `scripts/extract-facts.ts`).
- Section-level embeddings live in Turso table `chapter_embeddings`, lazy-loaded on first request (module-level cache).

## Tables (Turso)
- `anonymous_sessions` — cookie-keyed freemium tracking (≤5 questions).
- `users` — registered users (id, email, created_at, anonymous_session_id).
- `user_statistics` — per-user question log (question text + matched sources).
- `feedback` — thumbs up/down plus optional text, linked to question/answer/sources.
- `chapter_embeddings` — regulation, dir, file, section, label, text, embedding (JSON array).

## Scripts (`app/scripts/`)
- `generate-embeddings.ts` — compute section embeddings.
- `seed-embeddings.ts` — load embeddings into Turso (`npm run seed:embeddings`).
- `extract-facts.ts` — produce `facts.json`.
- `migrate.ts` — DB schema setup.
- `eval-quality.ts` — LLM-judge regression set.
- `analyze-feedback.ts`, `quality-scorecard.ts` — reporting.
- `run-e2e-tests.ts` — manual e2e runner.
- `generate-og-image.ts` — OG image asset.

## External dependencies (runtime)
- OpenRouter: `POST /api/v1/chat/completions` (streaming), `POST /api/v1/embeddings`.
- Auth0: OAuth code flow via `@auth/sveltekit` (`src/auth.ts`).
- Turso: libSQL over WebSocket.

## Environments
- dev: localhost (`vite dev`, port 5173), direct Turso cloud connection.
- prod: Vercel serverless, same Turso instance.

## Directory map
- `app/src/routes/` — SvelteKit routes (UI + API).
- `app/src/lib/server/` — server-only modules (`db.ts`, `queries.ts`, `router.ts`, `semantic-router.ts`, `chapters.ts`, `ai.ts`, `schema.sql`, `facts.json`).
- `app/src/lib/components/` — Svelte components.
- `app/knowledge/` — regulation markdown (loaded at runtime).
- `app/scripts/` — CLI maintenance scripts.
- `app/static/` — static assets.
- Top-level `k2-arsredovisning/`, `k3-arsredovisning/` etc. — source regulation extracts, synced into `app/knowledge/`.

## Notable constraints
- Stateless per request; no in-memory session (serverless friendly). Embedding cache is per-worker.
- Context cap: 80 000 chars (`chapters.ts` `MAX_CHARS`).
- Routing result cap: 4 chapters (semantic router). Keyword router is used both standalone (fallback) and as one RRF input.
- Answers generated with `temperature: 0`, `max_tokens: 2048`.
- Scope filter degrades gracefully: if no matches survive the scope filter, the unfiltered set is used rather than returning nothing.
